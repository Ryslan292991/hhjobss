# Поиск удалённых вакансий → Telegram

Каждые 3 часа GitHub сам запускает скрипт: тот идёт на hh.ru, забирает свежие удалённые вакансии без опыта по всей России, прогоняет через антифрод-фильтр и присылает выжившие тебе в личку.

Ничего не нужно устанавливать. Твой компьютер может быть выключен.

---

## Настройка: 15 минут, один раз

### 1. Создай бота

Напиши в Telegram [@BotFather](https://t.me/BotFather):

- `/newbot`
- придумай имя и username (должен кончаться на `bot`)

Он вернёт токен вида `7891234567:AAF-xxxxxxxxxxxxxxxx`. Сохрани.

### 2. Нажми Start у своего бота

Найди его по username и нажми кнопку **Start**. Без этого бот не имеет права тебе писать — Telegram запрещает первым писать незнакомым.

### 3. Узнай свой chat_id

Напиши [@userinfobot](https://t.me/userinfobot). Он пришлёт число вида `123456789`. Это и есть chat_id.

### 4. Создай репозиторий

Зайди на github.com → **New repository**.

Назови как хочешь. **Выбирай Public.** У приватных репозиториев бесплатно только 2000 минут Actions в месяц, а восемь прогонов в день их съедят. У публичных минуты не ограничены. Твои токены при этом в репозиторий не попадают — они лежат отдельно, в Secrets, и в коде их нет.

Загрузи туда все файлы из этой папки, вместе с папкой `.github`.

### 5. Пропиши секреты

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**.

Создай три штуки:

| Имя | Значение |
|---|---|
| `TELEGRAM_TOKEN` | токен из шага 1 |
| `TELEGRAM_CHAT_ID` | число из шага 3 |
| `HH_USER_AGENT` | `hh-jobs-bot/1.0 (твоя@почта.ru)` |

Последнее обязательно: hh требует, чтобы приложение представлялось, иначе банит запросы.

### 6. Проверь

Вкладка **Actions** → если GitHub спросит разрешения, нажми «I understand my workflows, go ahead and enable them» → слева выбери **Поиск вакансий** → кнопка **Run workflow**.

Через несколько минут в Telegram придут первые вакансии. Дальше всё поедет само по расписанию.

---

## Что отсеивается

**Жёстко, по названию:** колл-центры и контакт-центры во всех написаниях, операторы звонков, холодный обзвон, телемаркетинг, подбор персонала, рекрутинг.

**По баллам — антифрод.** Каждый признак добавляет вес. Сумма 5+ — вакансия выбрасывается молча. 3–4 — приходит с пометкой «⚠️ проверь внимательно» и списком причин. Что считается признаком:

- названия из скам-категорий: чат-менеджер, оператор переписки, наборщик текста, тайный покупатель, финансовый агент
- в тексте: взносы и залоги, платное обучение, ежедневные выплаты, «без вложений», «не сетевой маркетинг», крипта и ставки, признаки интим-подработки
- связь только через личный телеграм
- работодатель не верифицирован hh или скрыт за анонимной вакансией
- зарплата от 150 000 ₽ при нулевом опыте
- очень короткое описание при высокой зарплате

Пороги — в `config.py`: `SCAM_DROP_THRESHOLD` и `SCAM_WARN_THRESHOLD`.

## Как править под себя

Всё в `config.py`, менять прямо через веб-интерфейс GitHub (карандашик над файлом):

- `QUERIES` — поисковые запросы. Обычный текст, как в строке поиска hh.
- `PERIOD_DAYS` — за сколько дней брать вакансии.
- `MAX_SEND_PER_RUN` — потолок сообщений за прогон. Что не влезло, придёт в следующий раз, ничего не теряется.

Расписание — в `.github/workflows/find-jobs.yml`, строка `cron: "0 */3 * * *"`. Хочешь пореже: `"0 */6 * * *"`.

## Проверить, не режет ли фильтр лишнего

Вкладка **Actions** → любой прогон → шаг «Искать вакансии». В логе строки `DROP` показывают, что выброшено и почему. Первую неделю стоит заглядывать. Увидишь живую вакансию среди отброшенных — убери слово из `TITLE_STOPWORDS` в `filters.py` или подними `SCAM_DROP_THRESHOLD` с 5 до 6.

## Что может сломаться

**GitHub усыпляет расписание.** Если в репозитории 60 дней нет активности от человека, планировщик отключается — коммиты самого бота не считаются. GitHub пришлёт письмо. Достаточно зайти и что-нибудь поправить.

**hh отдаёт 403.** Поиск работает анонимно, но hh просит регистрировать приложения и может закрутить гайки. Бот тогда пришлёт сообщение с объяснением. Лечится так: регистрируешь приложение на [dev.hh.ru/admin](https://dev.hh.ru/admin), получаешь токен, добавляешь четвёртым секретом с именем `HH_TOKEN`. Код подхватит сам.

Не используй утилиты, которые притворяются мобильным приложением hh, чтобы обойти ограничение. Это нарушает условия API и ведёт к бану аккаунта.

**Cron опаздывает.** GitHub запускает по расписанию с задержкой 5–15 минут, иногда больше. Для поиска работы это не важно.

## Файлы

| Файл | Зачем |
|---|---|
| `run.py` | Основной скрипт. Один прогон — и выходит. |
| `filters.py` | Стоп-слова и антифрод. Самое интересное здесь. |
| `config.py` | Все настройки. |
| `seen.json` | Память: какие вакансии уже показывали. Бот коммитит его сам. |
| `.github/workflows/find-jobs.yml` | Расписание для GitHub. |
