# Поиск удалённых вакансий с нескольких сайтов → Telegram

Каждые 3 часа GitHub сам запускает бота. Он одновременно обходит несколько источников вакансий, сводит их в один поток, прогоняет через антифрод-фильтр и присылает выжившие тебе в Telegram — с пометкой, откуда каждая.

Ничего устанавливать не нужно, компьютер может быть выключен.

## Источники

| Источник | Статус | Комментарий |
|---|---|---|
| **Работа России** (trudvsem.ru) | основной | Государственный портал, открытый API. Работает всегда, ботов не блокирует. Работодатели проверены по ИНН/ОГРН. |
| **hh.ru** | резервный | С конца 2025 hh часто блокирует запросы с серверов (403). Бот это переживает: если hh недоступен, просто работает на остальных и одной строкой отмечает, что hh молчал. |

Устроено так, что добавить новый источник — это один файл в папке `sources/`. Захочешь — подключим ещё.

### Честно про Trudvsem

Госпортал силён по регионам и рабочим специальностям, а вот удалённого контента (копирайтинг, SMM) там меньше, чем на hh. Поэтому связка и собрана из двух источников: Trudvsem даёт стабильный поток и не банится, hh — когда доступен — добавляет офисно-цифровые вакансии. Если hh совсем перестанет отвечать, стоит подключить третий источник.

## Настройка: один раз, ~15 минут

### 1. Бот в Telegram
Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → получи токен вида `7891234567:AAF...`. **Никому не показывай.**

### 2. Нажми Start у своего бота
Найди бота по username, нажми **Start**. Без этого Telegram не даст ему тебе писать.

### 3. Узнай chat_id
Напиши [@userinfobot](https://t.me/userinfobot) — пришлёт число. Это chat_id (только цифры).

### 4. Репозиторий
github.com → New repository → назови как хочешь → **Public** (у публичных Actions бесплатны без лимита) → Create.
Загрузи все файлы, **включая папку `sources` и папку `.github`**.

Папки через перетаскивание не грузятся. Для каждого файла в подпапке используй **Add file → Create new file** и введи путь целиком, например `sources/trudvsem.py` — слэш сам создаст папку. Файлы: `sources/__init__.py` (пустой), `sources/base.py`, `sources/hh.py`, `sources/trudvsem.py`, и `.github/workflows/find-jobs.yml`.

### 5. Секреты
Settings → Secrets and variables → Actions → New repository secret. Создай три:

| Name | Secret |
|---|---|
| `TELEGRAM_TOKEN` | токен из шага 1 |
| `TELEGRAM_CHAT_ID` | число из шага 3 |
| `HH_USER_AGENT` | `jobs-bot/1.0 (твоя@почта.ru)` |

### 6. Разрешить запись
Settings → Actions → General → внизу **Workflow permissions** → **Read and write permissions** → Save. Без этого бот не сохранит память и будет слать одно и то же.

### 7. Запуск
Вкладка **Actions** (верхнее меню) → **Поиск вакансий** → **Run workflow**. Через несколько минут придут первые вакансии.

## Что отсеивается

**Жёстко, по названию:** колл-центры, операторы звонков, обзвон, телемаркетинг, подбор персонала, рекрутинг.

**По баллам — антифрод.** Сумма 5+ — вакансия выбрасывается молча, 3–4 — приходит с пометкой «⚠️ проверь». Признаки: скам-названия (чат-менеджер, наборщик текста, тайный покупатель), в тексте — взносы, платное обучение, ежедневные выплаты, «без вложений», крипта и ставки, интим-подтекст, связь только через личный телеграм, неверифицированный или анонимный работодатель, зарплата 150k+ без опыта.

Пороги — в `config.py`.

## Как править под себя

Всё в `config.py`, редактируется прямо на GitHub (карандашик над файлом):

- `SOURCES` — включить/выключить источники. Надоел шум про hh — поставь `"hh": False`.
- `QUERIES` — поисковые запросы, обычный текст.
- `MAX_SEND_PER_RUN` — потолок сообщений за прогон. Лишнее не теряется, придёт в следующий.
- `PERIOD_DAYS`, пороги антифрода.

Расписание — в `.github/workflows/find-jobs.yml`, строка `cron`.

## Проверить, не режет ли лишнего

Actions → любой прогон → шаг «Искать вакансии» → строки `DROP` показывают отброшенное с причиной и источником. Первую неделю стоит поглядывать.

## Что может сломаться

- **hh отдаёт 403** — это ожидаемо, не поломка. Бот работает дальше на Trudvsem. Убрать заметку — `SOURCES["hh"] = False`.
- **Trudvsem изменил формат ответа** — бот пришлёт «Источник trudvsem упал с ошибкой …». Скинь мне текст, поправлю парсер.
- **GitHub усыпляет расписание** после 60 дней без активности человека. Придёт письмо, достаточно зайти и что-нибудь поправить.
- **Мало удалённого контента** — добавь запросы в `QUERIES` или подключим третий источник.
